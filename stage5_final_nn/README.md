# Этап 5. Создание финальной версии НС

Эта папка оформлена для проверки этапа по регламенту. Здесь лежат два ноутбука и все файлы, которые нужны для их запуска.

## Что запускать

1. `01_final_ready_model.ipynb`  
   Финальный ноутбук: используется уже готовая модель. В конце есть тестовая ячейка, которая запускает весь pipeline: загрузка данных, очистка, подготовка признаков, создание loader, загрузка модели, prediction, postprocessing и понятный вывод `LONG / SHORT / WAIT`.

2. `02_experiments_model_building.ipynb`  
   Ноутбук с экспериментами: показано, как модель создавалась и сравнивалась. Внутри есть baseline, Logistic Regression, Random Forest, HistGradientBoosting, графики, метрики, выбор лучшей модели, сохранение модели и отдельный блок с более сильной TradeForge-моделью/backtest.

## Файлы рядом

- `btcusdt_15m_sample.csv` — sample данных BTCUSDT 15m для запуска ноутбуков.
- `trade_signal_model.joblib` — сохраненная готовая модель для первого ноутбука.
- `experiment_summary.csv` — краткая таблица результатов экспериментов.
- `top_tradeforge_model/` — дополнительные артефакты топовой TradeForge-модели и отчеты backtest.
- `stage4_project_report.docx` — Word-документ по проекту для приложения к этапу.

## Как проверить

Откройте ноутбуки в GitHub или Google Colab и запустите ячейки сверху вниз. Все нужные файлы лежат в этой же папке, отдельно скачивать ничего не нужно.

Если проверка идет в Colab, удобнее скачать папку целиком или загрузить файлы рядом с ноутбуком в рабочее пространство Colab.
